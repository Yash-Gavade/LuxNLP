# 📝 Reporting Module

The `reporting/` directory contains scripts for **exporting results, generating reports, and saving experiment outputs** in the LuxNLP pipeline.

This module transforms evaluation outputs and dataset statistics into **structured formats (CSV, tables, summaries)** suitable for analysis, visualization, and research documentation.

---

## 🎯 Purpose

The reporting module is used to:

* Export dataset statistics
* Save evaluation results in tabular form
* Store training configurations for reproducibility
* Generate structured model performance reports

👉 It acts as the bridge between **raw evaluation results and final presentation (tables, papers, dashboards)**.

---

## 📂 Files Overview

### `export_conll_stats_to_csv.py`

Exports CoNLL dataset statistics to CSV format.

**Includes:**

* Sentence counts
* Token counts
* Entity distribution

👉 Useful for dataset analysis and reporting.

---

### `export_eval_table.py`

Exports evaluation results into structured tables.

**Includes:**

* Precision
* Recall
* F1-score
* Per-entity performance

👉 Used for:

* Paper tables
* Model comparison
* Result summaries

---

### `export_training_args.py`

Saves training configuration parameters.

**Includes:**

* Hyperparameters
* Model settings
* Training arguments

👉 Important for:

* Reproducibility
* Experiment tracking

---

### `make_model_reports.py`

Generates complete model performance reports.

**Combines:**

* Evaluation metrics
* Dataset statistics
* Training details

👉 Produces a **comprehensive summary of model performance**.

---

## 🔄 Role in Pipeline

```text
Evaluation → Reporting → Visualization → Documentation
```

This module converts computed results into **usable outputs for analysis and presentation**.

---

## ⚙️ Example Usage

```bash
# Export dataset statistics
python scripts/Analysis/reporting/export_conll_stats_to_csv.py

# Export evaluation table
python scripts/Analysis/reporting/export_eval_table.py

# Save training configuration
python scripts/Analysis/reporting/export_training_args.py

# Generate full report
python scripts/Analysis/reporting/make_model_reports.py
```
