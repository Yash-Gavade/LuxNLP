# Predictions

This folder contains token-level Named Entity Recognition (NER) predictions generated by different models in CoNLL format.

Each file includes BIO-tagged outputs used for evaluation, comparison, and error analysis.

---

## File Format

All prediction files follow the **CoNLL format**, where each line represents a token and its predicted label:

```id="4k2mns"
Token    Predicted_Label
```

* Sentences are separated by empty lines
* Labels follow the BIO scheme (e.g., `B-PER`, `I-LOC`, `O`)

---

## Files

### Main Test Set Predictions

* **`xlmr_test_predictions.conll`**
  Predictions from the fine-tuned XLM-RoBERTa model on the main test dataset.
  These results represent the primary evaluation of the supervised model.

* **`groq_rag_test_predictions.conll`**
  Predictions from the Groq-based RAG approach on the main test dataset.
  Used for comparison with XLM-R performance.

---

### Survey Dataset Predictions (External Evaluation)

* **`xlmr_survey_predictions.conll`**
  Raw XLM-RoBERTa predictions on the survey dataset.
  Reflects performance under domain shift.

* **`xlmr_survey_predictions_filtered.conll`**
  Filtered version of XLM-R predictions on the survey dataset.
  Filtering reduces noise and slightly improves precision.

* **`groq_rag_survey_predictions.conll`**
  RAG model predictions on the survey dataset.
  Used to evaluate generalization and behavior on unseen data.

---

## Purpose

These prediction files are used for:

* Computing evaluation metrics (precision, recall, F1 score)
* Generating confusion matrices
* Error analysis and qualitative inspection
* Comparing supervised (XLM-R) and retrieval-based (RAG) approaches

---

## Key Observations

* XLM-RoBERTa predictions are more consistent and structured across both datasets.
* RAG predictions tend to over-predict entities, leading to higher recall but lower precision.
* On the survey dataset, both models show reduced performance due to domain differences.

---

## Notes

* The **main test set** follows the same distribution as the training data.
* The **survey dataset** is external and not used during training.
* These prediction files directly correspond to the evaluation results provided in the `metrics/` folder.

---

## Usage

You can use these files to:

* Recompute evaluation metrics
* Visualize model performance
* Analyze prediction errors
* Compare entity-level outputs across models

---

## Summary

This folder provides the model outputs that support the quantitative evaluation and comparison presented in the project. It enables full transparency and reproducibility of the results.
